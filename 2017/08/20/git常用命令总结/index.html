git常用命令总结

loner11
2017-08-20
Git

git init 初始化创建本地仓库
git clone git@... 克隆远程仓库内容至本地

git status 查看当前版本库所有状态
git diff 查看当前版本库修改
git log (--pretty=oneline参数) 查看当前版本库所有提交日志
git stash -> git pull origin master -> git stash pop 将当前修改暂时放在暂存区，然后拉取最新更新，避免多人协作发生冲突

完整git操作（从远程仓库拉取更新->更新本地仓库->修改本地仓库当前版本内容->提交本地当前版本更新至远程仓库）

  1. git clone git@... 将远程仓库内容克隆至本地
  2. git pull origin master 更新本地仓库
  3. git add filename (git add . 添加所有修改文件) 将filename文件添加至本地仓库的暂存区
  4. git commit -m 'content' 将暂存区内容提交至本地当前版本HEAD
  5. git push origin master 将本地当前更新版本内容提交至远程仓库默认名为origin的master分支

版本回退

HEAD当前版本HEAD^上一个版本HEAD^^上上一个版本HEAD~100

使用git reset命令回退版本 git reset --hard HEAD^

管理修改

撤销修改

git checkout --file 把文件在工作区的修改全部撤销（文件没有放入暂存区）回到工作区状态
git reset HEAD file 已经添加到暂存区，又做了修改，撤销修改回到和添加暂存区后的状态

删除文件

rm file 删除工作区文件
git rm file 删除版本库中的文件

添加远程仓库（多人协作）

git remote add origin git@... 添加远程仓库默认名为origin
git remote -v 显示所有远程仓库

首次推送本地仓库内容至远程仓库

git push -u origin master 第一次提交
git push origin master 本地提交

创建分支

git checkout -b dev 创建并切换至dev分支
git branch dev 创建dev分支
git checkout dev 切换至dev分支
git merge dev 合并dev分支到当前分支
git log --graph 查看分支合并图
git branch -d dev 删除分支

注意: 合并分支时，如果可能Git会用Fast forward模式，但是这种情况下，删除分支后，会丢掉分支信息

Git自定义配置

git config --global user.name 'name' 配置用户名
git config --global user.email 'email' 配置用户邮箱
git config --global color.ui true 配置git显示颜色

忽略特殊文件

.gitignore —> https://github.com/github/gitignore

忽略文件原则

  1. 忽略操作系统自动生成的文件
  2. 忽略编译生成的中间文件、可执行文件
  3. 忽略你自己的带有敏感信息的配置文件

添加已经被忽略的文件到Git

使用-f强制添加到git git add -f ignore.js
使用命令检查忽略规则 git check-ignore
